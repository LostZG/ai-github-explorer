# Product Guidelines - AI GitHub Explorer

## 1. 文档风格规范
- **语言风格**：采用专业且严谨的中文风格编写项目文档和代码注释，确保术语准确、简练。
- **排版建议**：统一使用现代无衬线字体（如 Inter, PingFang SC），确保在不同设备上的优秀阅读体验。

## 2. 视觉设计与交互标准
- **核心色调**：以深邃黑为底，主打电光蓝/紫（Electric Blue/Purple）色调，营造现代感和活力。
- **组件风格**：卡片应具备柔和的渐变发光边缘和微妙的投影效果，提升科技氛围。
- **交互规范**：鼠标悬浮在卡片上时应有 1.05 倍的细腻缩放动画，并伴随柔和的辉光（Glow）特效。

## 3. 开发与代码质量控制
- **Git 提交**：严格遵守 Conventional Commits 规范（如 `feat:`, `fix:`, `docs:`）。
- **工具链集成**：项目应默认集成 Prettier 和 ESLint，通过自动化工具确代码格式高度一致。
- **模块化架构**：每个逻辑功能模块（如 GitHub API 抓取器、UI 组件库）应具备独立的命名空间和清晰的目录结构。

## 4. 文档维护与更新
- **功能详述**：详细记录 GitHub 项目的抓取逻辑、缓存策略以及瀑布流的布局算法。
- **入门流程**：维护一份极其简洁表明的 `Quick Start` 指南，确保护航开发者在 3 分钟内完成环境搭建。
- **贡献准则**：为外部开发者提供明确的 Pull Request 流程和代码审查标准。

## 5. 交互原则
- **响应速度优先**：所有交互动作必须在 100ms 内给出视觉反馈。
- **内容为王**：UI 装饰不应干扰用户对 GitHub 项目标题和概述的快速扫描。
